Overview

A single source of truth for your design system. Export tokens from Figma, transform them into platform-native code using Style Dictionary.

◈

Token Source

Tokens are authored in JSON files following the W3C Design Token Community Group (DTCG) format, compatible with Figma Variables and Tokens Studio exports.

⚙

Transform

Style Dictionary v4 reads the token JSON and transforms values — resolving references, converting units, and applying platform-specific naming conventions.

↗

Output

Single build command generates CSS, SCSS, Android XML, iOS Swift, and TypeScript — all synchronized from the same token source.

Token Format (W3C DTCG)

All token files use the W3C Design Token Community Group format. Each token has a $value and $type:

// tokens/primitives/color.json
{
  "color": {
    "blue": {
      "500": {
        "$value": "#3b82f6",
        "$type": "color",
        "$description": "Core brand blue"
      }
    }
  }
}

// Semantic tokens can reference primitives:
{
  "semantic": {
    "color": {
      "action": {
        "primary": {
          "$value": "{color.blue.500}",
          "$type": "color"
        }
      }
    }
  }
}

Colors

A comprehensive 6-hue palette with 10 shades each, designed for accessible contrast ratios and harmonious visual hierarchy.

Typography

Font families, size scale, weights, line heights, and letter spacing — all tokenized for cross-platform consistency.

Spacing

A 4px base-unit spacing scale. Every value is a multiple of 4px for consistent, predictable layouts.

Border Radius

Corner radius tokens from sharp to full-pill, applied consistently across buttons, cards, inputs, and more.

Shadows

Elevation shadows for conveying depth and hierarchy. Each shadow level has a specific use case.

Opacity

Opacity scale for overlays, disabled states, and subtle layering effects.

Semantic Colors

Role-based color aliases that reference primitive tokens. These are the tokens your components should use — they're swappable for theming.

.button-primary {

}

.input {
  border: 1px solid var(--semantic-color-border-default);
  border-radius: var(--border-radius-md);
  padding: var(--spacing-2) var(--spacing-3);
  font-family: var(--font-family-sans);
  font-size: var(--font-size-base);

SCSS Guide

Generated as SCSS variables with optional reference output. Integrates naturally into existing Sass-based design systems.

Output filesbuild/scss/_variables.scss, _map.scss
Formatsscss/variables, scss/map-deep
Naming$token-path-kebab-case

Import

// Using @use (recommended for Sass modules)
@use '../build/scss/variables' as tokens;

.button {
  background: tokens.$semantic-color-action-primary;
}

// Or with namespace wildcard
@use '../build/scss/variables' as *;

.button {
  background: $semantic-color-action-primary;
}

// Using the map (for programmatic access)
@use '../build/scss/map' as *;
@use 'sass:map';

$primary: map.get($tokens, 'semantic', 'color', 'action', 'primary');

Generated Output Sample

// _variables.scss
$color-blue-50: #eff6ff;
$color-blue-500: #3b82f6;
$color-blue-600: #2563eb;
// ... all primitives ...

// Semantic tokens with SCSS variable references:
$semantic-color-action-primary: $color-blue-600;
$semantic-color-text-primary: $color-gray-900;

Usage Examples

@use '../build/scss/variables' as *;

// Component mixin using tokens
@mixin card($variant: 'default') {
  background: $semantic-color-surface-default;
  border: 1px solid $semantic-color-border-default;
  border-radius: $border-radius-lg;
  padding: $spacing-6;
  box-shadow: $shadow-md;

  @if $variant == 'raised' {
    box-shadow: $shadow-lg;
  }
}

// Typography mixin
@mixin text-style($size, $weight: 'regular') {
  font-family: $font-family-sans;
  font-size: map.get($font-size, $size);
  font-weight: map.get($font-weight, $weight);
  line-height: $font-line-height-normal;
}

// Generating utility classes
@each $name, $value in (
  'primary': $semantic-color-action-primary,
  'secondary': $semantic-color-action-secondary,
  'error': $semantic-color-feedback-error
) {
  .bg-#{$name} { background-color: $value; }
  .text-#{$name} { color: $value; }
}

Android Guide

Generated as Android XML resource files. Colors use ARGB hex format, dimensions are converted to dp/sp units.

Namingtoken_path_snake_case

Setup

Copy the generated XML files into your Android project's app/src/main/res/values/ directory:

app/
└── src/
    └── main/
        └── res/
            └── values/
                ├── colors.xml    ← from build/android/colors.xml
                └── dimens.xml    ← from build/android/dimens.xml

Usage in Kotlin

// Access color resource
val primaryColor = ContextCompat.getColor(context, R.color.color_blue_600)
view.setBackgroundColor(primaryColor)

// Access dimension resource
val spacing = resources.getDimensionPixelSize(R.dimen.spacing_4)
view.setPadding(spacing, spacing, spacing, spacing)

// In Jetpack Compose — use colorResource and dimensionResource
@Composable
fun PrimaryButton(text: String, onClick: () -> Unit) {
    Button(
        onClick = onClick,
        colors = ButtonDefaults.buttonColors(
            containerColor = colorResource(R.color.color_blue_600),
            contentColor = colorResource(R.color.color_white)
        ),
        contentPadding = PaddingValues(
            horizontal = dimensionResource(R.dimen.spacing_6),
            vertical = dimensionResource(R.dimen.spacing_3)
        )
    ) {
        Text(text)
    }
}

Automating the Copy Step

Add a script to package.json to copy generated files into your Android project after build:

// package.json
{
  "scripts": {
    "build:android": "node sd.config.js && cp build/android/*.xml ../android-app/app/src/main/res/values/"
  }
}

Usage in UIKit

import UIKit

class PrimaryButton: UIButton {
    override init(frame: CGRect) {
        super.init(frame: frame)
        configure()
    }

    private func configure() {
        backgroundColor = StyleDictionaryColor.colorBlue600
        setTitleColor(StyleDictionaryColor.colorWhite, for: .normal)
        titleLabel?.font = .systemFont(
            ofSize: StyleDictionarySize.fontSizeSm,
            weight: .semibold
        )
        layer.cornerRadius = StyleDictionarySize.borderRadiusMd
        contentEdgeInsets = UIEdgeInsets(
            top: StyleDictionarySize.spacingThree,
            left: StyleDictionarySize.spacingSix,
            bottom: StyleDictionarySize.spacingThree,
            right: StyleDictionarySize.spacingSix
        )
    }
}

Usage in SwiftUI

import SwiftUI

extension Color {
    static let tokenBlue600 = Color(StyleDictionaryColor.colorBlue600)
    static let tokenGray900 = Color(StyleDictionaryColor.colorGray900)
    static let tokenWhite   = Color(StyleDictionaryColor.colorWhite)
}

struct PrimaryButton: View {
    let title: String
    let action: () -> Void

    var body: some View {
        Button(action: action) {
            Text(title)
                .font(.system(size: StyleDictionarySize.fontSizeSm, weight: .semibold))
                .foregroundColor(.tokenWhite)
                .padding(.horizontal, StyleDictionarySize.spacingSix)
                .padding(.vertical, StyleDictionarySize.spacingThree)
                .background(Color.tokenBlue600)
                .cornerRadius(StyleDictionarySize.borderRadiusMd)
        }
    }
}

// Or use directly with Color(uiColor:) in newer SwiftUI
Text("Error")
    .foregroundColor(Color(uiColor: StyleDictionaryColor.colorRed600))

Figma Export Guide

How to export tokens from Figma into this system's JSON format using Figma Variables or the Tokens Studio plugin.

Option 1 — Tokens Studio Plugin

Tokens Studio (formerly Figma Tokens) is the most popular token management plugin for Figma.

  1. Install Tokens Studio for Figma from the Figma Community
  2. Define your tokens in the plugin panel, organized into sets (Primitives, Semantic, etc.)
  3. Click Export → Export to JSON (W3C Format)
  4. Place exported files in the tokens/ directory, replacing the sample files
  5. Run npm run build
Note: Tokens Studio exports use the W3C DTCG format with $value and $type — exactly what this system expects.

Option 2 — Figma Variables (REST API)

Figma's native Variables feature can be exported via the REST API:

# Get variables from Figma file
curl -H "X-Figma-Token: YOUR_PERSONAL_ACCESS_TOKEN" \
  "https://api.figma.com/v1/files/YOUR_FILE_ID/variables/local"

The response requires a transformation script to convert Figma's variable format to W3C DTCG JSON. A community tool like figma-variables-to-dtcg can handle this.

Option 3 — Manual JSON

Edit the token JSON files directly. The format is simple:

{
  "color": {
    "brand": {
      "primary": {
        "$value": "#0066cc",   // The token value
        "$type": "color",       // Token type
        "$description": "Primary brand color"  // Optional
      }
    }
  }
}

Supported Token Types

$typeUsed forExample value
colorColors"#3b82f6"
dimensionSizes, spacing, radii"16px"
fontFamilyFont stacks"Inter, sans-serif"
fontWeightFont weights"600"
numberUnitless numbers1.5
shadowBox shadows{ offsetX, offsetY, blur, spread, color }

Reference Syntax

Reference another token by its dot-separated path in curly braces:

{
  "semantic": {
    "color": {
      "action": {
        "primary": {
          "$value": "{color.blue.600}",  // References color → blue → 600
          "$type": "color"
        }
      }
    }
  }
}
